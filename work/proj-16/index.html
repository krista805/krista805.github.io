Crypto Portfolio Tracker using Vue/Nuxt, bootstrap, and the CoinGecko API.

My main goal for this project was to get a better familiarity with componentization. Although I work primarily on a Vue application for my current job, I am following already established patterns for our components.

The first step I took was to create user stories, divided into two phases. The first phase was user stories required to complete in order to launch. The second phase was more “nice to have”, but not necessary features. I found that dividing it into phases was helpful because it felt feasable to complete a smaller batch of stories. Plus it kept me from spending time on the ‘nice to haves’. Once I had the user stories, I mocked a basic wireframe of the UI. Next, I sliced up the wireframe, splitting it into the core components: Portfolio Summary, AssetList, and AssetInput.

Within each component, I created a markdown file with the associated user stories Here is an example of the AssetList components' markdown file:

  • As a user, I can see a list of the crypto assets I added
  • I can view the following in the table row
    • Asset name, ticker symbol, crypto logo
    • Amount in crypto
    • Current value in USD
    • Allocation in this asset (based on USD value)
  • I can filter the table to show my crypto assets by coin name or by USD value
  • I have the option to delete any crypto I may add to this list

Learning experience:

  • You don’t need to componentize everything. It makes it harder to pass data between parent and sibling components
  • Don’t get caught up in the designs. I put this project off for a while because I didn’t have a design implementation.
  • There are several ways to go about doing things in Vue. Be open to going back and refactoring your code.
